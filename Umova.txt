Квітковий магазин.
Визначити ієрархію квітів.
Створити декілька об'єктів-квітів.
Зібрати букет (використовуючи аксесуари) з визначенням його вартості.
Провести сортування квітів у букеті на основі рівня свіжості.
Знайти квітку в букеті, що відповідає заданому діапазону довжин стебел.

Використовувати можливості ООП: класи, успадкування, поліморфізм, інкапсуляція.
Кожен клас повинен мати змістовну назву та інформативне наповнення.
Успадкування повинно застосовуватися лише тоді, коли це має сенс.
При кодуванні використати оформлення коду відповідно до Java Code Convention.
https://www.oracle.com/docs/tech/java/codeconventions.pdf
Класи повинні бути коректно розподілені по Java-пакетах.
Робота з консоллю або консольне меню повинно бути мінімальним.
Для збереження параметрів ініціалізації можна використовувати файли.
Весь код повинен бути покритий Unit-тестами.
Використовувати JUnit 4 або 5, Mockito.
Код повинен бути розміщений на віддаленому репозиторії.
Історія змін повинна мати логічні commit-повідомлення.
Репозиторій має мати правильно налаштований gitignore файл.
